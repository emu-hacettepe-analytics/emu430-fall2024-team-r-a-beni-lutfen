The Code:

Fetching data using quantmod

prices_lst <- list()

  • API Stability: We replaced the TCMB EVDS API with quantmod (Yahoo Finance). EVDS frequently suffers from SSL/TLS handshake failures and connection timeouts on Windows.

  • Consistency: Fetching all assets (including the USD/TRY exchange rate TRY=X) from a single global source ensures identical timestamps and database formats.

  • 10-Year Horizon: A longer timeframe captures multiple macroeconomic cycles and structural shifts. Yahoo Finance automatically back-adjusts BIST prices for the 2020 denomination (two-zero drop) reform, preserving historical continuity.

2. Multi-Market Synchronization (The inner_join Strategy)

The Code:

Merging multiple datasets into a single tibble

price_data <- prices_lst %>% reduce(inner_join, by = “Date”)

  • Calendar Mismatch: Forex (TRY=X) trades 24/5 globally and remains open during Turkish local or religious holidays, while Borsa Istanbul (BIST) strictly closes.
  • The Solution: We used inner_join to keep only the dates common to all assets. If BIST was closed on a holiday, that day is discarded. This ensures our statistical models compare active trading days only.

3. Handling Missing Values (drop_na) & Outlier Philosophy

The Code:

Ensuring a clean, gap-free matrix

price_data <- price_data %>%

drop_na()

  • No Imputation: In finance, filling missing values using averages (imputation) or carrying prices forward is highly discouraged. It artificially dampens volatility and biases daily returns. Removing incomplete rows with drop_na() is the standard, unbiased approach.
  • Keeping Outliers: While statistical tests flag extreme returns (shocks) as outliers, we did not delete them. In finance, outliers represent real market crises and circuit-breakers. Removing them would destroy the exact macro-shocks we aim to study.

4. Mathematical Transformation: Logarithmic Returns

The Code:
Transforming raw prices to stationary log returns

return_data <- price_data %>% mutate(across(starts_with(“Raw_”), ~ log(.x / lag(.x)), .names = “Ret_{.col}”)) %>% rename_with(~ gsub(“Ret_Raw_”, “Ret_”, .x)) %>% drop_na()

Stationarity: Raw asset prices have random walks and trends, violating the core assumptions of regression and correlation. Logarithmic returns (\(R_t = \ln(P_t / P_{t-1})\)) stabilize mean and variance, making the time series stationary.

  • Time Additivity: Unlike simple percentage returns, the sum of daily log returns over a period exactly equals the total log return of that entire period.

  • Symmetry: Log returns are symmetric. A \(10\%\) gain followed by a \(10\%\) loss returns the asset to its exact starting value, preventing statistical skewness.

5. Multi-Scale Normalization (Cumulative Base-100 Returns)

The Code:

pd_norm <- pd %>% mutate(across(-Date, ~ (.x / first(.x)) * 100))

  • Scale Incompatibility: You cannot visually compare USD/TRY (ranging between 1.5 to 32+ TL) with the BIST 100 index (ranging in thousands of points) on the same Y-axis.

  • The Solution: We normalized all assets to a Base-100 index starting on the first day of each period. This scales all asset growth relative to a common starting line (\(Price_{t_0} = 100\)), enabling a fair, percentage-based visual comparison.

6. Currency Denomination (USD-Denominated BIST)

The Code:

BIST100_USD = XU100.IS / TRY=X

  • Eliminating the Inflation Illusion: In high-inflation economies, stock indices in local currency (TRY) almost always trend upward. However, these “rallies” are often nominal illusions driven by currency depreciation.

  • The Solution: Dividing BIST prices by the USD/TRY rate reveals the USD-denominated (real) value of Turkish equities. This shows whether the market actually grew in global purchasing power or simply inflated.
